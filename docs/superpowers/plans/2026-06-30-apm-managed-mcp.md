# APM-Managed MCP Configuration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Manage this repo's GitHub and IDE/code-navigation MCP servers through `apm.yml`, then let APM generate the client-specific project config for the repo's supported agent targets.

**Architecture:** `apm.yml` is the only hand-maintained source of truth for MCP server declarations. `apm install` generates the project-scoped runtime config files for Claude, Codex, Copilot/VS Code, and OpenCode, while `AGENTS.md` and README stay human-facing and `.github/copilot-instructions.md` remains a minimal pointer to `AGENTS.md`.

**Tech Stack:** APM (`apm.yml`, `apm install`), GitHub MCP registry package (`io.github.github/github-mcp-server`), Serena MCP registry package (`oraios/serena`), JSON/TOML/YAML config files

---

````md
## MCP configuration

Project-scoped MCP servers are declared in `apm.yml` under `dependencies.mcp`.

- Do not hand-maintain parallel MCP definitions for Copilot, Claude, Codex, or OpenCode when APM can generate them.
- After changing `dependencies.mcp`, run `apm install` to refresh the project config files.
- Keep secrets out of committed files. The GitHub MCP install expects `token` at install time, and the Serena server requires local `uvx`.
````
